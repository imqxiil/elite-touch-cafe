## API ENDPOINTS
- GET list endpoints → correct data structure returned
- GET single item → correct item, 404 on invalid ID
- POST create → creates record, returns new item with ID
- PUT/PATCH update → updates correctly, returns updated item
- DELETE → deletes record, returns 200/204
- All responses have correct HTTP status codes
- Error responses consistent: { error: string, code: number }

## DATA VALIDATION
- Missing required fields → expect 400/422
- Wrong data types → expect 400
- SQL injection strings → must not crash or expose data
- XSS payloads (<script>alert(1)</script>) → must be sanitized
- Oversized payload (10MB) → expect 413

## AUTH & AUTHORIZATION
- No token → expect 401
- Invalid/expired token → expect 401
- Access another user's private resource → expect 403
- Admin-only endpoints reject non-admin users → expect 403
- JWT tokens expire correctly

## DATABASE INTEGRITY
- Duplicate unique records → expect 409 Conflict
- Cascading deletes work (parent delete removes children)
- No N+1 query issues on list endpoints

## PERFORMANCE
- API response times < 500ms for standard reads
- Large dataset queries use pagination, not full table scan
- No memory leaks under rapid consecutive requests

## EDGE CASES
- Empty database state → list returns [] not error
- Single record → works correctly
- 1000+ records → pagination works, no timeout
- Concurrent requests (same resource) → no race condition
- Negative numbers / decimals / zero in numeric fields
- Past / future / invalid dates in date fields
